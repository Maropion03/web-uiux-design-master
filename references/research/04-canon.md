## A. 必读经典书（13 本）

### A1. 时代经典层（≥10 年长红，几乎每份 canon 书单都在）

#### 1. The Design of Everyday Things — Don Norman ⭐⭐⭐
- 中译：《设计心理学 1：日常的设计》
- 一句话：以人为中心的设计圣经；提出 affordance / signifier / mental model / mapping 这套行业共同语言
- 为什么必读：Norman 在原版（1988）即创造了"User Experience"一词；2013 修订版补充了 signifier 概念，仍是 IxDF / Coursera / 几乎所有「UX must-read」书单第 1 名
- 是否过时：**否**。例子换皮但模型不变
- 适合阶段：入门第 1 本
- 来源：[IxDF UX Books Guide](https://ixdf.org/literature/article/ux-design-books-guide), [Coursera UX Resource List](https://www.coursera.org/articles/ux-design-books-blogs-podcasts), [UX Design Institute](https://www.uxdesigninstitute.com/blog/must-read-books-for-ux-designers/)

#### 2. Don't Make Me Think (Revisited) — Steve Krug ⭐⭐⭐
- 中译：《点石成金：访客至上的网页设计秘笈》
- 一句话：网页可用性的最小公约数手册——"显而易见"是终极目标
- 为什么必读：3rd ed (2014) 是 Web 可用性的入门快餐；NN/g、Smashing、Coursera、UX Beginner 全部点过
- 是否过时：略——例子仍是桌面 web；原则不过时
- 适合阶段：入门第 2 本（一个周末读完）
- 来源：[Coursera](https://www.coursera.org/articles/ux-design-books-blogs-podcasts), [IxDF](https://ixdf.org/literature/article/ux-design-books-guide), [UXfolio Top 15](https://blog.uxfol.io/ux-design-books/)

#### 3. About Face: The Essentials of Interaction Design (4th ed., 2014) — Alan Cooper, Reimann, Cronin, Noessel ⭐⭐⭐
- 一句话：交互设计学科教科书；提出 persona / goal-directed design 方法论
- 为什么必读：交互设计学科的奠基者亲自写的最系统化教材；4th ed 加入触屏与移动平台
- 是否过时：方法论不过时，移动端章节略旧（2014）
- 适合阶段：进阶（想做交互设计 / 产品设计师方向）
- 来源：[Wiley official](https://www.wiley.com/en-us/About+Face:+The+Essentials+of+Interaction+Design,+4th+Edition-p-9781118766576), [IxDF](https://ixdf.org/literature/article/ux-design-books-guide)

#### 4. Information Architecture: For the Web and Beyond (4th ed., 2015) — Rosenfeld, Morville, Arango ⭐⭐⭐
- 别名："Polar Bear Book" / "IA 圣经"
- 一句话：信息架构正典；首版 1998，售出 25 万+ 册
- 为什么必读：IA 这个子领域基本是这本书定义的
- 是否过时：4th ed 已加入"the web and beyond"（mobile / cross-channel）
- 适合阶段：要做大型站点 / 内容系统时必读
- 来源：[Polar Bear Book official](https://intertwingled.org/the-polar-bear-book/), [O'Reilly](https://www.oreilly.com/library/view/information-architecture-4th/9781491913529/)

#### 5. Thinking with Type (3rd ed., 2024) — Ellen Lupton ⭐⭐⭐
- 一句话：视觉传达类型学的入门权威，三段结构（letter / text / grid）
- 为什么必读：所有 UI / 平面 / 网页设计师的字体启蒙书；Lupton 是 MICA / Cooper Hewitt 教授
- 是否过时：**否**，2024 年第 3 版刚出
- 适合阶段：视觉方向必读，交互方向选读
- 来源：[Ellen Lupton official](https://ellenlupton.com/Thinking-with-Type), [PA Press](https://papress.com/products/thinking-with-type-3-edition)

#### 6. 100 Things Every Designer Needs to Know About People — Susan Weinschenk ⭐⭐
- 一句话：把认知心理学翻译成 100 条可操作的设计 tip
- 为什么必读：碎片式学，最好用的 UX 心理学厕所读物；多份书单点过
- 适合阶段：入门 / 平时翻
- 来源：[IxDF](https://ixdf.org/literature/article/ux-design-books-guide), [Team W Top 10](https://www.blog.theteamw.com/2013/01/23/top-10-best-usability-ux-and-design-books-you-should-read/)

#### 7. The Elements of User Experience (2nd ed., 2010) — Jesse James Garrett ⭐⭐
- 一句话：UX 五层模型（strategy → scope → structure → skeleton → surface）
- 为什么必读：很多公司面试还在用这套词汇；模型本身是初学者的脚手架
- 是否过时：略——模型偏 waterfall，但仍是行业通用语
- 适合阶段：入门，建立 UX 思维框架
- 来源：[UX Beginner reading list](https://uxbeginner.com/ux-reading-list/), [UX Booth](https://uxbooth.com/articles/user-experience-books-for-beginners/)

### A2. Web/UI 实操经典层

#### 8. Refactoring UI — Adam Wathan & Steve Schoger ⭐⭐⭐
- 一句话：开发者也能立刻上手的 UI 视觉技巧 cheat sheet
- 为什么必读：Tailwind 作者写的；2018+ 几乎成为「程序员转 UI」的标配；Tailwind UI 直接体现书里的原则
- 是否过时：**否**，2026 年仍是最高 ROI 的 UI 视觉书
- 适合阶段：所有 web 方向开发/设计师
- 来源：[refactoringui.com](https://refactoringui.com/), [Nucleus UI 2024 list](https://www.nucleus-ui.com/blog/best-ui-design-books-2024)

#### 9. Atomic Design — Brad Frost (2016, 免费在线) ⭐⭐⭐
- 一句话：把 UI 拆成 atoms / molecules / organisms / templates / pages 的心智模型
- 为什么必读：design system 思潮的开山书；术语已成行业默认词汇
- 是否过时：**否**，但 2026 年实践已被 design tokens 补强
- 适合阶段：做组件库 / design system 前必读
- 来源：[atomicdesign.bradfrost.com](https://atomicdesign.bradfrost.com/), [designsystems.com](https://www.designsystems.com/brad-frosts-atomic-design-build-systems-not-pages/)

#### 10. Design Systems — Alla Kholmatova (Smashing, 2017) ⭐⭐
- 一句话：基于 AirBnB / Atlassian / TED 案例研究，讲怎么建立「共享语言」式 design system
- 为什么必读：Atomic Design 偏理论，这本偏实操与组织
- 来源：[designsystemsbook.com](https://designsystemsbook.com/), [Smashing](https://www.smashingmagazine.com/design-systems-book/)

#### 11. Design That Scales — Dan Mall (Rosenfeld, 2024) ⭐⭐
- 一句话：design system 的「运营」而非「构建」——怎么让 system 活下来
- 为什么必读：2024 年新经典；Dan Mall 是最早 design system 顾问之一
- 适合阶段：已经有 design system，想 sustainable 化
- 来源：[Rosenfeld Media](https://rosenfeldmedia.com/books/design-that-scales/), [Design System University](https://designsystem.university/books/design-that-scales)

### A3. 心理学/行为设计层

#### 12. Laws of UX — Jon Yablonski (O'Reilly, 2020；2nd ed. 2024) ⭐⭐⭐
- 一句话：把 21 条心理学定律（Fitts / Hicks / Gestalt / Jakob / Miller / Peak-End 等）翻译给 UX
- 为什么必读：配套 lawsofux.com 已成 onboard 新人时的标配链接；2024 年出了第二版
- 适合阶段：入门到中级
- 来源：[lawsofux.com](https://lawsofux.com/), [Looppanel 21 Laws](https://www.looppanel.com/blog/laws-of-ux)

#### 13. Deceptive Patterns — Harry Brignull (2023) ⭐⭐
- 一句话：dark patterns 这个词的发明者 13 年研究的集大成（更名 deceptive patterns）
- 为什么必读：2023 年立法浪潮（GDPR / FTC）让这本书从「设计伦理」变成「合规必读」
- 适合阶段：所有 senior，特别是做电商 / 订阅产品的
- 来源：[deceptive.design](https://www.deceptive.design/book), [Wikipedia Dark Pattern](https://en.wikipedia.org/wiki/Dark_pattern)

### A3b. 近 3 年新经典候补
- **Lean UX (3rd ed., 2021) — Gothelf & Seiden** ⭐⭐：agile 团队里怎么做 UX；偏团队/流程
- **The Making of a Manager — Julie Zhuo (2019)** ⭐⭐：前 FB 设计副总裁，更偏「设计经理」而非纯 UX，但 design lead 圈高频引用
- **A Project Guide to UX Design — Unger & Chandler** ⭐⭐：年轻设计师入行 second-pick

## D. 核心 25 个概念词

「这行的人脑子里的正典词汇」。每条 = 1 句话定义 + 出处。

### D1. Norman 体系（认知/交互基础）
1. **Affordance（可供性）** — 物品本身允许的动作可能性。 *出处：Gibson (生态心理学) → Norman 《Design of Everyday Things》*
2. **Signifier（指示器）** — 让用户**感知**到 affordance 的线索。 *Norman 2013 修订版正式区分 affordance 与 signifier*
3. **Mental Model（心智模型）/ Conceptual Model** — 用户脑中对系统怎么运作的图。设计目标是让 system model 匹配 user mental model。 *Norman*
4. **Mapping（映射）** — 控件位置与功能之间的对应关系（炉灶旋钮经典案例）。 *Norman*
5. **Feedback（反馈）** — 用户动作后系统立刻可见地回应。 *Norman / Nielsen Heuristic #1*
6. **Constraints（约束）** — 物理/逻辑/文化/语义上阻止错误操作的设计。 *Norman*

### D2. Nielsen 体系（可用性）
7. **Usability Heuristics（10 条启发式）** — Nielsen 1994 的可用性十诫。 *NN/g*
8. **Heuristic Evaluation（启发式评估）** — 用 10 条标准评 UI 的轻量方法。 *Nielsen-Molich 1990 CHI*
9. **Jakob's Law** — 用户花大多数时间在别的网站上，希望你的网站像别人的。 *Jakob Nielsen / Laws of UX*

### D3. 阅读与扫描模式
10. **F-Pattern** — 用户阅读文本密集网页的 F 形轨迹。 *NN/g 2006 / 2017 mobile follow-up*
11. **Information Scent（信息气味）** — 用户判断链接是否能通向目标信息的线索强度。 *Peter Pirolli & Stuart Card, Xerox PARC, Information Foraging Theory*

### D4. 心理学定律
12. **Fitts's Law** — 击中目标的时间 ∝ log(距离/尺寸)。 *Fitts 1954*
13. **Hick's Law** — 决策时间随选项数对数增长。 *Hick 1952 / Hyman 1953*
14. **Miller's Law (7±2)** — 短期记忆能 hold 7±2 个 chunk。 *George Miller 1956*
15. **Peak-End Rule** — 体验由高峰与结尾决定。 *Kahneman*
16. **Gestalt Principles** — proximity / similarity / closure / continuity / figure-ground 等视觉分组律。 *Gestalt 心理学派 1920s*
17. **Cognitive Load** — 工作记忆负担；UI 应最小化外在认知负荷。 *Sweller*

### D5. 视觉/排版
18. **Visual Hierarchy** — 通过 size / weight / color / spacing 引导注意流。 *Refactoring UI 主题*
19. **Typographic Scale** — 系统化字号比例（如 1.25 / 1.333 / 1.618）。 *Lupton, Refactoring UI*
20. **Grid System** — 列/baseline 的对齐结构。 *Müller-Brockmann → Lupton*

### D6. 系统化与工程
21. **Atomic Design** — atoms → molecules → organisms → templates → pages。 *Brad Frost 2016*
22. **Design Tokens** — 把颜色/间距/字号等设计决策存为命名变量，跨平台分发。 *Salesforce Lightning / Nathan Curtis*
23. **Semantic Color** — 用语义命名（`color-danger`）而非具体值（`red-500`）。 *Design tokens 命名法 / Nathan Curtis*
24. **Responsive Design** — 用 fluid grid + media query + flexible image 适配多屏。 *Ethan Marcotte, A List Apart 2010*

### D7. 伦理/合规
25. **Dark Patterns / Deceptive Patterns** — 利用认知偏差欺骗用户的 UI 套路（roach motel, confirmshaming, drip pricing, etc）。 *Harry Brignull 2010 / 2023 book*
26. **WCAG (A / AA / AAA)** — Web Content Accessibility Guidelines 三级合规标准。 *W3C WAI*
27. **A11y** — accessibility 缩写（a + 11 letters + y）

### D8. 流程
28. **Design Thinking（五步：Empathize → Define → Ideate → Prototype → Test）** — *Stanford d.school / IDEO*
29. **Double Diamond** — Discover → Define → Develop → Deliver。 *UK Design Council 2005*
30. **Persona / Goal-Directed Design** — *Alan Cooper, About Face*

## 矛盾与争议保留（不和稀泥）

- **Hooked vs Deceptive Patterns**：Nir Eyal 的 Hook 模型被部分设计伦理派（包括 Brignull）批评为「dark patterns 武器化」；Eyal 后来出《Indistractable》算半个回应。设计师自己选立场。
- **Atomic Design vs Design Tokens 派**：2016 Atomic Design 偏组件层级；2020+ design tokens / W3C DTCG（2025 稳定版）派认为应从 token 而非组件出发。两者互补但侧重不同。
- **Google UX Cert vs IxDF**：前者短平快 + 大厂背书；后者广度深度都更深，但慢。Reddit /r/userexperience 长期辩论。
- **About Face 的 persona 方法**在 Lean UX / Jobs to be Done 派看来过于重，Cooper 2014 之后没更新。

---

## 缺信息维度（信心不足）

1. **中文圈一手原创书**：研究有意避开了知乎/公众号，但因此中文圈本土 UI/UX 经典几乎没有抓到样本——可能是真的没有同等量级的中文原创经典（多为译本），也可能是搜索策略偏向英文圈。建议下一轮特别针对**张小龙微信公开演讲**、**俞军《产品方法论》**、**唐韧博客**等做一手抓取。
2. **2024-2026 新锐书**：除 Design That Scales (Dan Mall, 2024) 和 Deceptive Patterns (2023) 外，没找到其他近 3 年挤进 canon 的新书——可能 canon 本身就更新很慢，也可能漏抓。
3. **AI × UX 的 canonical 书**还没成型；目前以 NN/g 文章 + Smashing 长文为主。这是 2026 视角下最大的内容缺口。
4. **HCI 学术经典**（Bill Buxton《Sketching User Experiences》、Saul Greenberg 等）未充分覆盖；学界/业界 canon 部分重合但不完全一致。
